<PageDescription>

Design tokens are the visual design atoms of the ELEVATE Design System. They store design decisions as data, enabling consistent implementation across different platforms, tools, and codebases.

</PageDescription>

## What are Design Tokens?

Design tokens are name-value pairs that represent design decisions. Instead of hardcoding values like `#6366f1` or `16px`, we use semantic names like `--elevate-primary` or `--elevate-space-md`.

### Benefits of Design Tokens

- **Consistency** – Shared values ensure visual consistency across products
- **Maintainability** – Update tokens in one place to change across all implementations
- **Scalability** – Easy to extend and modify as the system grows
- **Platform Agnostic** – Works with CSS, JavaScript, iOS, Android, and more

### Typography Tokens

```css
/* Font families */
--elevate-font-family-sans: 'Inter', sans-serif;
--elevate-font-family-mono: 'JetBrains Mono', monospace;

/* Font sizes */
--elevate-font-size-xs: 0.75rem;   /* 12px */
--elevate-font-size-sm: 0.875rem;  /* 14px */
--elevate-font-size-md: 1rem;      /* 16px */
--elevate-font-size-lg: 1.125rem;  /* 18px */
--elevate-font-size-xl: 1.25rem;   /* 20px */

/* Font weights */
--elevate-font-weight-normal: 400;
--elevate-font-weight-medium: 500;
--elevate-font-weight-semibold: 600;
--elevate-font-weight-bold: 700;
```

### Spacing Tokens

Our spacing system follows a consistent scale based on multiples of 4px:

```css
--elevate-space-xs: 0.25rem;  /* 4px */
--elevate-space-sm: 0.5rem;   /* 8px */
--elevate-space-md: 1rem;     /* 16px */
--elevate-space-lg: 1.5rem;   /* 24px */
--elevate-space-xl: 2rem;     /* 32px */
--elevate-space-2xl: 3rem;    /* 48px */
--elevate-space-3xl: 4rem;    /* 64px */
```

## Usage Examples

### CSS Implementation

```css
.button-primary {
  background-color: var(--elevate-primary);
  color: var(--elevate-text-inverse);
  padding: var(--elevate-space-sm) var(--elevate-space-md);
  font-family: var(--elevate-font-family-sans);
  font-weight: var(--elevate-font-weight-medium);
  border-radius: var(--elevate-border-radius);
}
```

### JavaScript Implementation

```js
import { tokens } from '@elevate/design-tokens';

const buttonStyles = {
  backgroundColor: tokens.color.primary,
  padding: `${tokens.space.sm} ${tokens.space.md}`,
  fontSize: tokens.fontSize.md,
  fontWeight: tokens.fontWeight.medium,
};
```

## Token Naming Convention

Our tokens follow a consistent naming pattern:

```
--elevate-[category]-[property]-[variant]
```

Examples:
- `--elevate-color-primary` – Primary brand color
- `--elevate-color-text-secondary` – Secondary text color
- `--elevate-space-md` – Medium spacing value
- `--elevate-border-radius-sm` – Small border radius

## Getting Started

### Installation

```bash
npm install @elevate/design-tokens
```

### Basic Usage

```css
@import '@elevate/design-tokens/css/variables.css';

.my-component {
  color: var(--elevate-color-text-primary);
  background: var(--elevate-color-background);
  padding: var(--elevate-space-md);
}
```

<InlineNotification>

**Pro tip:** Always use design tokens instead of hardcoded values to ensure your implementations stay consistent with the design system.

</InlineNotification>
