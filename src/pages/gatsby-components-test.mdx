## Interactive Components

### Tabbed Content

<Tabs>

<Tab label="Overview">

Tabs organize related content into separate views, allowing users to focus on specific information without overwhelming the interface.

<Code className="language-html">
<elevate-tabs>
  <elevate-tab label="Tab 1">Content 1</elevate-tab>
  <elevate-tab label="Tab 2">Content 2</elevate-tab>
</elevate-tabs>
</Code>

</Tab>

<Tab label="Best Practices">

- Use clear, descriptive tab labels
- Limit to 2-7 tabs for optimal usability
- Ensure content is logically organized
- Consider mobile responsiveness

</Tab>

<Tab label="Accessibility">

- Full keyboard navigation support
- ARIA labels and roles implemented
- Screen reader announcements
- Focus management across tabs

</Tab>

</Tabs>

### Accordion

<Accordion>
<AccordionItem title="What is the ELEVATE Design System?">

ELEVATE is INFORM's comprehensive design system that provides reusable components, design tokens, and guidelines for building consistent user experiences across all INFORM products.

</AccordionItem>
<AccordionItem title="How do I get started with ELEVATE components?">

Install the `@inform-elevate/elevate-core-ui` package and import the components you need. All components are built as web components, so they work with any framework or vanilla JavaScript.

</AccordionItem>
<AccordionItem title="Are ELEVATE components accessible?">

Yes! All ELEVATE components are built with accessibility as a core requirement, meeting WCAG 2.1 AA standards and supporting keyboard navigation, screen readers, and high contrast modes.

</AccordionItem>
</Accordion>
