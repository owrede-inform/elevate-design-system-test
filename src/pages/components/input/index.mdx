## Variations

### Input Types
The Input component supports all standard HTML5 input types:
- **Text** - Default single-line text input
- **Email** - Email address input with validation
- **Password** - Secure input with masked characters
- **Number** - Numeric input with step controls
- **Search** - Search input with enhanced functionality
- **Tel** - Telephone number input
- **URL** - URL input with validation
- **Date** - Date picker input
- **Time** - Time picker input
- **DateTime-Local** - Combined date and time picker

### Input Sizes
- **Small** - Compact input for dense layouts
- **Medium** - Default size for most use cases
- **Large** - Larger input for emphasis or accessibility

### Additional Features
- **Clearable** - Add clear button when value is present
- **Prefix/Suffix slots** - Icons or text before/after input
- **Form integration** - Native HTML form validation support
- **Autocomplete** - Browser autocomplete suggestions
- **Validation states** - Visual feedback for form validation

## Design Guidelines

### When to Use
- **Form fields**: Registration, login, profile editing
- **Search functionality**: Site search, filtering, lookup
- **Data entry**: Numeric values, dates, measurements
- **User feedback**: Comments, reviews, messages

### When Not to Use
- **Multi-line text**: Use Textarea component instead
- **Selection from options**: Use Select, Radio, or Checkbox
- **File uploads**: Use File Dropzone component
- **Toggle states**: Use Switch component

### Label Guidelines
- Always provide clear, descriptive labels
- Use sentence case for labels
- Keep labels concise (1-3 words when possible)
- Place labels above inputs for optimal scanning

### Validation Feedback
- Show validation in real-time when appropriate
- Use clear, actionable error messages
- Provide positive feedback for successful completion
- Don't clear valid data when showing errors

## Code Reference

### Installation

```bash
npm install @inform-elevate/elevate-core-ui
```

### Basic Usage

```html
<elvt-input 
  type="email"
  placeholder="Enter your email"
  value="">
  <label slot="label">Email Address</label>
</elvt-input>
```

### With Validation and Helper Text

```html
<elvt-input 
  type="password"
  required
  minlength="8">
  <label slot="label">Password</label>
  <span slot="help-text">Password must be at least 8 characters</span>
</elvt-input>
```

### With Prefix and Suffix Icons

```html
<elvt-input type="search" clearable>
  <label slot="label">Search</label>
  <elvt-icon slot="prefix" icon="mdi:magnify"></elvt-icon>
</elvt-input>
```

### Different Sizes

```html
<elvt-input size="small" placeholder="Small input"></elvt-input>
<elvt-input size="medium" placeholder="Medium input"></elvt-input>
<elvt-input size="large" placeholder="Large input"></elvt-input>
```

### Number Input with Constraints

```html
<elvt-input 
  type="number" 
  min="0" 
  max="100" 
  step="5" 
  value="25">
  <label slot="label">Percentage</label>
</elvt-input>
```

### With Autocomplete

```html
<elvt-input 
  type="text" 
  autocomplete="given-name"
  autocapitalize="words">
  <label slot="label">First Name</label>
</elvt-input>
```

### Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `type` | `InputType` | `text` | Input type: `text`, `email`, `password`, `number`, `search`, `tel`, `url`, `date`, `time`, `datetime-local` |
| `value` | `string` | - | Current input value |
| `placeholder` | `string` | - | Placeholder text shown when empty |
| `size` | `Size` | `medium` | Input size: `small`, `medium`, `large` |
| `disabled` | `boolean` | `false` | Whether the input is disabled |
| `readonly` | `boolean` | `false` | Whether the input is read-only |
| `required` | `boolean` | `false` | Whether the input is required |
| `clearable` | `boolean` | `false` | Add clear button when value is present |
| `autocomplete` | `string` | - | Browser autocomplete attribute |
| `autocapitalize` | `AutoCapitalize` | - | Auto-capitalize behavior: `off`, `none`, `on`, `sentences`, `words`, `characters` |
| `spellcheck` | `boolean` | - | Enable spell checking |
| `inputmode` | `InputMode` | - | Virtual keyboard type hint |
| `enterkeyhint` | `EnterKeyHint` | - | Enter key label hint for virtual keyboards |
| `autofocus` | `boolean` | `false` | Automatically focus on page load |
| `min` | `number \| string` | - | Minimum value (for number/date inputs) |
| `max` | `number \| string` | - | Maximum value (for number/date inputs) |
| `minlength` | `number` | - | Minimum character length |
| `maxlength` | `number` | - | Maximum character length |
| `pattern` | `string` | - | Regular expression pattern for validation |
| `step` | `number` | - | Step value for number inputs |

### Slots

| Slot | Description |
|------|-------------|
| `label` | Input label content |
| `help-text` | Helper text content |
| `prefix` | Icon or content before the input |
| `suffix` | Icon or content after the input |

### Events

| Event | Description |
|-------|-------------|
| `input` | Fired when input value changes (real-time) |
| `change` | Fired when input loses focus with changed value |
| `focus` | Fired when input receives focus |
| `blur` | Fired when input loses focus |

### Form Integration

The Input component extends `CustomFormField` and provides:
- Native HTML form validation
- Integration with form submission and reset
- Automatic form value management
- Support for validation constraints
- Element internals for accessibility

## Accessibility

### Keyboard Support
- **Tab**: Moves focus to the input
- **Shift + Tab**: Moves focus away from the input
- **Arrow keys**: Navigate within text (standard text editing)
- **Enter**: Submits form (if applicable)

### Screen Reader Support
- Labels are properly associated with inputs using `for` attribute
- Required fields are announced to screen readers
- Validation messages are announced when they appear
- Helper text is associated with the input

### Focus Management
- Clear focus indicators that meet WCAG contrast requirements
- Focus remains on input during validation
- Focus moves logically through form fields

### Best Practices
- Always provide labels for all inputs
- Use appropriate input types for better mobile keyboards
- Ensure error messages are descriptive and actionable
- Maintain sufficient color contrast (4.5:1 minimum)
- Don't rely solely on color to indicate validation states
