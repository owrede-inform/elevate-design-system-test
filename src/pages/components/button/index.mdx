## Variations

The Button component supports multiple visual tones and shapes to establish clear hierarchy and communicate different action types.

### Tone Variations
- **Neutral** - Default button for standard actions
- **Primary** - Main action on a page or workflow (use sparingly)
- **Success** - Positive actions like save, confirm, or complete
- **Warning** - Caution actions that need attention
- **Danger** - Destructive actions like delete or remove
- **Emphasized** - High-emphasis actions with strong visual weight
- **Subtle** - Low-emphasis actions with minimal visual treatment

### Shape Variations
- **Box** (default) - Standard rectangular button
- **Pill** - Rounded button with full corner radius

### Size Variations
- **Small** - Compact buttons for dense layouts
- **Medium** - Default size for most use cases
- **Large** - Prominent buttons for hero sections

## Design Guidelines

### When to Use
- **Primary actions**: Sign up, submit, save, confirm
- **Secondary actions**: Cancel, back, learn more  
- **Navigation**: Next step, continue, proceed
- **Destructive actions**: Delete, remove, reset

### When Not to Use
- **Links**: Use Link component for navigation between pages
- **Toggles**: Use Switch component for on/off states
- **Selection**: Use Checkbox or Radio for multiple choice

### Button Hierarchy
Establish clear visual hierarchy using button variants:
1. **Primary**: Most important action (1 per page)
2. **Secondary**: Supporting actions (2-3 per page)
3. **Tertiary**: Least important actions

### Sizing Guidelines
- **Large**: Hero sections, primary CTAs
- **Medium**: Default size for most use cases  
- **Small**: Compact layouts, data tables

## Code Reference

### Installation

```bash
npm install @inform-elevate/elevate-core-ui
```

### Basic Usage

```html
<elvt-button>Default Button</elvt-button>
<elvt-button tone="primary">Primary Button</elvt-button>
<elvt-button tone="success">Success Button</elvt-button>
<elvt-button tone="danger">Danger Button</elvt-button>
```

### With Icons (Prefix/Suffix Slots)

```html
<elvt-button tone="primary">
  <elvt-icon slot="prefix" icon="mdi:plus"></elvt-icon>
  Add Item
</elvt-button>

<elvt-button tone="secondary">
  Continue
  <elvt-icon slot="suffix" icon="mdi:chevron-right"></elvt-icon>
</elvt-button>
```

### Different Shapes and Sizes

```html
<elvt-button shape="pill" size="large" tone="primary">Pill Button</elvt-button>
<elvt-button size="small" tone="subtle">Small Button</elvt-button>
```

### As Link

```html
<elvt-button href="https://example.com" target="_blank">
  External Link
</elvt-button>
```

### Form Integration

```html
<elvt-button type="submit" tone="primary">Submit Form</elvt-button>
<elvt-button type="reset" tone="secondary">Reset Form</elvt-button>
```

### With Custom Padding

```html
<elvt-button padding="0 xl">Custom Padding</elvt-button>
```

### Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `tone` | `ButtonTone` | `neutral` | Visual tone: `neutral`, `primary`, `success`, `warning`, `danger`, `emphasized`, `subtle` |
| `size` | `Size` | `medium` | Button size: `small`, `medium`, `large` |
| `shape` | `string` | `box` | Button shape: `box`, `pill` |
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `selected` | `boolean` | `false` | Whether the button appears selected |
| `type` | `ButtonType` | `button` | Button type: `button`, `submit`, `reset` |
| `href` | `string` | - | When set, renders as `<a>` tag with this URL |
| `target` | `LinkTarget` | - | Link target (when `href` is used) |
| `rel` | `string` | `noreferrer noopener` | Link relationship (when `href` is used) |
| `download` | `string` | - | Download filename (when `href` is used) |
| `name` | `string` | - | Form field name |
| `value` | `string` | - | Form field value |
| `padding` | `Padding` | - | Custom padding using design token values |

### Slots

| Slot | Description |
|------|-------------|
| `default` | The button's label text |
| `prefix` | Icon or element before the label |
| `suffix` | Icon or element after the label |

### Events

| Event | Description |
|-------|-------------|
| `click` | Fired when button is clicked (includes form submission logic) |
| `focus` | Fired when button receives focus |
| `blur` | Fired when button loses focus |

### Methods

| Method | Description |
|--------|-------------|
| `focus(options?: FocusOptions)` | Programmatically focus the button |

## Accessibility

### Keyboard Support
- **Space/Enter**: Activates the button
- **Tab**: Moves focus to the button
- **Shift + Tab**: Moves focus away from the button

### Screen Reader Support
- Buttons have proper `role="button"` semantics
- Button text is announced to screen readers
- Disabled state is communicated to assistive technology
- Loading state provides appropriate feedback

### Focus Management
- Clear focus indicators meet WCAG contrast requirements
- Focus is trapped appropriately in modal contexts
- Focus returns to appropriate element after button actions

### Best Practices
- Use descriptive button text that clearly indicates the action
- Avoid generic text like "Click here" or "Learn more"
- Ensure sufficient color contrast (4.5:1 minimum)
- Maintain minimum touch target size of 44px Ã— 44px
- Provide alternative text for icon-only buttons
