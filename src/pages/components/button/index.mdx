## Design Guidelines

### When to Use
- **Primary actions**: Sign up, submit, save, confirm
- **Secondary actions**: Cancel, back, learn more  
- **Navigation**: Next step, continue, proceed
- **Destructive actions**: Delete, remove, reset

### When Not to Use
- **Links**: Use Link component for navigation between pages
- **Toggles**: Use Switch component for on/off states
- **Selection**: Use Checkbox or Radio for multiple choice

### Button Hierarchy

<DoDontRow>
<DoDontExample type="do" caption="Use one primary button per page for the main action">

### Properties

<PageTable>

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `tone` | `ButtonTone` | `neutral` | Visual tone: `neutral`, `primary`, `success`, `warning`, `danger`, `emphasized`, `subtle` |
| `size` | `Size` | `medium` | Button size: `small`, `medium`, `large` |
| `shape` | `string` | `box` | Button shape: `box`, `pill` |
| `disabled` | `boolean` | `false` | Whether the button is disabled |
| `selected` | `boolean` | `false` | Whether the button appears selected |
| `type` | `ButtonType` | `button` | Button type: `button`, `submit`, `reset` |
| `href` | `string` | - | When set, renders as `<a>` tag with this URL |
| `target` | `LinkTarget` | - | Link target (when `href` is used) |
| `rel` | `string` | `noreferrer noopener` | Link relationship (when `href` is used) |
| `download` | `string` | - | Download filename (when `href` is used) |
| `name` | `string` | - | Form field name |
| `value` | `string` | - | Form field value |
| `padding` | `Padding` | - | Custom padding using design token values |

</PageTable>

### Slots

| Slot | Description |
|------|-------------|
| `default` | The button's label text |
| `prefix` | Icon or element before the label |
| `suffix` | Icon or element after the label |

### Events

| Event | Description |
|-------|-------------|
| `click` | Fired when button is clicked (includes form submission logic) |
| `focus` | Fired when button receives focus |
| `blur` | Fired when button loses focus |

### Methods

| Method | Description |
|--------|-------------|
| `focus(options?: FocusOptions)` | Programmatically focus the button |

## Accessibility

### Keyboard Support
- **Space/Enter**: Activates the button
- **Tab**: Moves focus to the button
- **Shift + Tab**: Moves focus away from the button

### Screen Reader Support
- Buttons have proper `role="button"` semantics
- Button text is announced to screen readers
- Disabled state is communicated to assistive technology
- Loading state provides appropriate feedback

### Focus Management
- Clear focus indicators meet WCAG contrast requirements
- Focus is trapped appropriately in modal contexts
- Focus returns to appropriate element after button actions

### Best Practices
- Use descriptive button text that clearly indicates the action
- Avoid generic text like "Click here" or "Learn more"
- Ensure sufficient color contrast (4.5:1 minimum)
- Maintain minimum touch target size of 44px Ã— 44px
- Provide alternative text for icon-only buttons
