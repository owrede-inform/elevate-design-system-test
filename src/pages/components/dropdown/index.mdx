### When to Use Dropdowns
- **Form selections**: Country, state, category, and type selections
- **Filters and sorting**: Data table filters and sort options
- **Settings menus**: Configuration options and preferences
- **Action menus**: Context-specific actions and commands
- **Navigation**: Secondary navigation and menu structures

### When NOT to Use Dropdowns
- **Two options**: Use radio buttons or toggle switches for binary choices
- **Many options**: Consider autocomplete or search for 10+ options
- **Always visible info**: Use visible text for frequently referenced information
- **Primary navigation**: Use dedicated navigation components for main site sections

### Predictability Principle
**Users should understand what will happen when they interact with dropdowns.** Clear trigger styling and consistent behavior patterns help users predict dropdown functionality without trial and error.

### Efficiency Principle  
**Minimize steps and cognitive load for common selections.** Frequently chosen options should be easy to find and select. Long scrolling or complex navigation defeats the space-saving purpose of dropdowns.

### Clarity Principle
**Selected states and available options should be immediately obvious.** Users need to understand current selections and see all available choices clearly. Ambiguous states create uncertainty and selection errors.

### Responsive Adaptation Principle
**Dropdowns should work effectively across different screen sizes and input methods.** Mobile touch interfaces have different requirements than desktop mouse interaction.

### Organizing Options Logically

**Why this matters:** Random or poorly organized option lists force users to scan through irrelevant choices. Logical organization reduces cognitive load and helps users find their desired selection quickly.

### Handling Long Option Lists

**Why this matters:** Dropdowns with many options become unwieldy and hard to navigate. Users get overwhelmed with too many choices and may struggle to find what they need.

### Providing Clear Selection States

**Why this matters:** Users need confirmation of their selections and ability to understand current dropdown state. Unclear selection feedback creates uncertainty about form completion and data entry.

### Essential Accessibility Practices

**Keyboard navigation support:**
- Space/Enter opens dropdown menu
- Arrow keys navigate through options
- Escape closes dropdown menu
- Tab moves to next form element

**Screen reader compatibility:**
- Use proper ARIA roles (combobox, listbox, option)
- Announce current selection and available options
- Provide labels and descriptions for dropdown purpose
- Communicate expanded/collapsed states

**Focus management:**
- Clear focus indicators on trigger and options
- Logical focus order through dropdown interface
- Return focus to trigger when dropdown closes
- Highlight selected options appropriately

**Selection feedback:**
- Announce selection changes to screen readers
- Provide visual confirmation of selected options
- Clear indication of multi-selection states
- Error messages for validation failures
