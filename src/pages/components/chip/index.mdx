## Variations

### Input Chip
Represents user selections or entered data that can be removed.

### Filter Chip
Interactive chip used for filtering content with active/inactive states.

### Choice Chip
Selectable chip for single or multiple choice selections.

### Action Chip
Triggers actions when clicked, similar to a compact button.

### Avatar Chip
Chip with user avatar or icon for person-related actions.

### Status Chip
Indicates status or state with semantic colors and labels.

## Design Guidelines

### When to Use
- **Tags and labels**: Categorizing or labeling content
- **Active filters**: Showing applied search or content filters  
- **User selections**: Displaying chosen options or items
- **Compact actions**: Space-constrained interactive elements
- **Status indicators**: Quick status communication

### When Not to Use
- **Primary actions**: Use Button component for main interactions
- **Long text content**: Use appropriate text components
- **Complex information**: Use Card or other layout components
- **Navigation**: Use proper navigation components

### Content Guidelines
- Keep text short and scannable (1-3 words ideal)
- Use sentence case for better readability
- Be descriptive but concise
- Avoid technical jargon when possible
- Consider internationalization for text content

### Color and Styling
- Use semantic colors for status chips
- Maintain sufficient contrast for readability
- Keep styling subtle to avoid visual clutter
- Use consistent sizing within groups
- Consider accessibility in color choice

## Code Reference

### Installation

```bash
npm install @inform-elevate/elevate-core-ui
```

### Basic Usage

```html
<elvt-chip>Design System</elvt-chip>
<elvt-chip variant="outlined">Frontend</elvt-chip>
<elvt-chip variant="filled">React</elvt-chip>
```

### Dismissible Chip

```html
<elvt-chip dismissible>
  Removable Tag
</elvt-chip>
```

### Filter Chip

```html
<elvt-chip 
  variant="filter" 
  selected="true">
  Active Filter
</elvt-chip>
```

### With Icon

```html
<elvt-chip>
  <elvt-icon name="star"></elvt-icon>
  Favorite
</elvt-chip>
```

### Avatar Chip

```html
<elvt-chip>
  <elvt-avatar size="small" src="/user.jpg" alt="John Doe"></elvt-avatar>
  John Doe
</elvt-chip>
```

### Properties

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `variant` | `string` | `default` | Visual style: `default`, `outlined`, `filled`, `filter` |
| `size` | `string` | `medium` | Chip size: `small`, `medium`, `large` |
| `selected` | `boolean` | `false` | Whether the chip is selected (for filter/choice chips) |
| `disabled` | `boolean` | `false` | Whether the chip is disabled |
| `dismissible` | `boolean` | `false` | Whether the chip can be dismissed/removed |
| `clickable` | `boolean` | `false` | Whether the chip is clickable |
| `color` | `string` | `default` | Color variant: `default`, `primary`, `success`, `warning`, `error` |

### Events

| Event | Description |
|-------|-------------|
| `click` | Fired when chip is clicked (if clickable) |
| `dismiss` | Fired when chip is dismissed/removed |
| `selection-change` | Fired when chip selection state changes |

## Accessibility

### Keyboard Support
- **Tab**: Moves focus to interactive chips
- **Enter/Space**: Activates clickable chips or toggles selection
- **Delete/Backspace**: Removes dismissible chips when focused
- **Arrow keys**: Navigate between chips in groups

### Screen Reader Support
- Chips announce their content and state clearly
- Interactive chips have proper button or option semantics
- Selected state is communicated for filter/choice chips
- Dismissible action is clearly described

### Focus Management
- Clear focus indicators on interactive chips
- Focus moves logically through chip groups
- Focus is managed appropriately when chips are removed
- Initial focus placement follows expected patterns

### ARIA Support
- Uses appropriate roles (`button`, `option`, etc.) based on function
- Selected states use `aria-selected` or `aria-pressed`
- Dismissible chips have accessible remove actions
- Groups use proper `role="group"` when needed

### Best Practices
- Provide clear, descriptive chip text
- Ensure sufficient color contrast for all variants
- Use semantic roles appropriate to chip function
- Don't rely solely on color to convey information
- Provide keyboard alternatives for all mouse interactions
- Group related chips logically
